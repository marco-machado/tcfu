# UI primitive library and central style layers

Status: accepted
Date: 2026-07-15

The DOM UI is consolidated into a shadcn-style primitive library. Every reusable UI atom becomes one canonical React component under `src/app/components/ui/` (Button, IconButton, Panel, Heading, Label, Meter, PipRow, Chip, Toggle, Slider, Segmented, Modal, Overlay, ScreenHeader, plus the existing Icon and BrandMark). Screens compose these components and never hand-write design class strings. Variants are props mapped to modifier classes by a small internal `cn()` helper (`btn` plus `btn--primary`); component state stays on `is-*` classes, matching the existing CSS convention. Screen-specific compositions (HUD clusters like score/survival/weapons/wave/boss, and cards like bay-card/ship-card/kit-preview) stay in their screens, rebuilt on top of the primitives rather than promoted into the library.

As part of this, `src/shell/` is merged into `src/app/`: `app/screens/`, `app/hud/`, `app/components/ui/`, `app/components/` (non-primitive shared chrome: Stage, MenuBackdrop, DebugPanel), and `app/styles/`. The engine layers (`sim`, `view`, `presentation`, `audio`, `input`, `persist`) stay as top-level siblings. Styling lives in one central place, split into `app/styles/{tokens,base,primitives,screens}.css` and ordered with `@layer tokens, base, primitives, screens`. The clipped-corner notch collapses to a three-step token scale (`--notch-sm: 6px`, `--notch: 8px`, `--notch-lg: 10px`, default 8px), replacing the arbitrary 5/7/9 offsets. The token palette gains a small set of functional tokens (surface, border, and text tiers) so recurring greys and border rgbas have one source; hardcoded hexes in the primitive layer are replaced with tokens. The primitive catalog and per-component variant APIs live in `docs/design/UI_PRIMITIVES.md`.

This makes visual drift structural to prevent rather than a recurring cleanup: a finite variant set defined in one component, one notch scale, and one token source per role. It keeps the DOM UI (`app`) and the 3D scene (`view`) as distinct render targets, and leaves `CONTEXT.md` as a pure domain glossary (primitive names are implementation vocabulary and belong in the design docs, not the glossary). Definition of done is a full migration: every screen consumes `ui/` and the superseded ad-hoc classes are deleted, done incrementally with the app compiling and `vitest` green throughout, so no two parallel systems coexist at the end.

Rejected alternatives: Tailwind plus class-variance-authority to colocate styling in JSX like literal shadcn (a build-level migration that fights the existing token, clip-path, and container-query system for no consistency gain); per-component CSS modules (spreads styling across many files, the opposite of the single-source goal, and adds module tooling the project does not use); a single flat `components/` folder (erases the primitive-versus-composition line the library is built on); folding `sim`/`view`/`presentation` into `app` (destroys the render-only engine boundary from ADR 0006); and a flat 8px notch (uniform, but visibly shrinks the cut on large panels and enlarges it on chips versus today).
